進階挑戰 - Ollama Function Calling

Ollama 的新利器：Function Calling 讓 AI 更靈活強大

在July 25, 2024，Ollama發布了 Tool support 文章，介紹了 Tools (或稱 Function Calling)的功能，其中提到文章內容如下：

Ollama now supports tool calling with popular models such as Llama 3.1. This enables a model to answer a given prompt using tool(s) it knows about, making it possible for models to perform more complex tasks or interact with the outside world.
Example tools include:

  • Functions and APIs
  • Web browsing
  • Code interpreter
  • much more!

簡單來說，這個功能讓 Ollama 的模型能夠使用各種工具來回答問題或完成任務。

Ollama 提到的工具範例包括：

  • Function 和 API：模型可以調用預定義的函數或外部 API，例如獲取實時天氣數據或股票價格。
  • 網頁瀏覽：模型能夠訪問網頁內容，獲取最新的新聞或信息。
  • 代碼解釋器：允許模型執行和分析代碼片段，對於程序設計和調試很有幫助。

接下來，將通過實例來示範如何在我們的 FastAPI 專案中使用 Ollama 的 Function Calling 功能。

開放模型

開放使用 Tool 的常見模型如下：

  • Llama 3.1
  • Mistral Nemo
  • Firefunction v2
  • Command-R +

更多支援 Tools 的模型可以從 Ollama Models 清單（Tag 設置為 Tools）找到。

注意：請確保模型版本為最新，若非最新，使用指令拉取最新模型：ollama pull

實際開發

REST API Request

在開始實際開發之前，讓我們先來看看 Ollama 官方提供的 REST API 呼叫範例中的 tools 部分：

Request tools 結構：

  1. tools 是一個數組，其中包含多個工具定義。
  2. 每個工具有 type（目前為 “function”）和 function 參數。
  3. function 包含：
    • name: 函數名稱
    • description: 函數描述
    • parameters: 參數定義（使用 JSON Schema 格式）
💡

重要提示：使用 tools 時，stream 必須設為 false。

REST API Response

使用了 tools 的 API 呼叫可能會得到如下格式的回傳結果：

Response 結構：

  • message：包含 AI 助理的回應
    • role：固定為 “assistant”
    • content：通常為空字符串
    • tool_calls：包含函數調用的詳細信息
  • done 和 done_reason：指示請求完成狀態

程式碼解析

  1. 首先在此段程式碼中我們會進行第一次對模型的呼叫，在呼叫中會加入 tools 參數，並在tools 中描述 function 的描述以及必填參數，此處以 get_current_weather 作為範例，告訴 Model 此 Tool 可取得即時天氣。在呼叫模型後判斷是否成功並取得模型回傳內容。

  2. 接著創建一個 get_current_weather 方法，假裝可以取得指定地區的即時天氣資訊。

  3. 判斷第一次呼叫模型回傳的結果是否需要使用 Tools，如果 message 內不含 tool_calls 則不需要呼叫 tool 直接回傳，message 內含 tool_calls 則根據內容呼叫指定 function 與帶入傳遞參數。

  4. 最後進行第二次呼叫，將 tools 回傳結果再次回傳給模型，讓模型解析 tool 結果並回傳結果。
